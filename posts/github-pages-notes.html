从 0 到 GitHub Pages 的发布笔记

2026.06.07 3 min GitHub / 部署

这个站点是纯静态结构：HTML、CSS、JavaScript 和图片。它不依赖数据库，也不需要服务器进程，因此很适合放在 GitHub Pages 上。

上线需要三件事

第一，仓库里保留 CNAME 文件，内容是 blog.66zhang.cn。第二，在 GitHub Pages 里选择从 GitHub Actions 发布。第三，在域名 DNS 里把 blog 子域名指向 GitHub Pages。

为什么不用复杂框架

个人博客的第一目标是能持续写。静态页面的好处是迁移简单、加载快、几乎不会被依赖升级打断。

以后可以加什么

等文章数量变多后，可以再引入静态站生成器，把文章从 HTML 换成 Markdown。现在这个版本先让博客真正上线。
