技术知识库不是为了收藏资料，而是为了让下一次排障、部署或学习更快。记录得越结构化，复用价值越高。

## 从问题模板开始

每条技术笔记可以固定包含：问题现象、环境版本、关键命令、日志片段、排查路径、最终方案和后续注意事项。模板简单，长期积累才容易坚持。

```bash
systemctl status nginx
journalctl -u nginx --since "10 minutes ago"
curl -I https://blog.66zhang.cn
```

## 让标签服务检索

网络、Linux、云服务、安全、AI、工具这些标签不是装饰，而是检索入口。以后遇到类似问题，可以直接按主题回看。

## 保持轻量但可迁移

初期不需要复杂系统。先保证内容能发布、能修改、能搜索、能迁移，等文章变多后再引入 Markdown、构建脚本和全文索引。
